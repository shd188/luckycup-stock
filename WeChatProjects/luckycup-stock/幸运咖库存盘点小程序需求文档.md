# 幸运咖库存盘点小程序需求文档

------------------------------------------------------------------------

## 一、项目目标

开发一个用于幸运咖门店库存盘点的小程序，实现：

-   按总部固定分类结构展示物料
-   门店库存填写（允许为0，不允许为空）
-   提交后不可修改
-   支持公开分享
-   所有盘点数据持久化存储
-   为后续数据分析和功能升级预留扩展空间

------------------------------------------------------------------------

## 二、产品结构说明

### 1. 分类结构

固定为两级结构：

    分类A
        产品1
        产品2
    分类B
        产品3
        产品4

说明：

-   永远不会升级为三级结构
-   所有门店共用同一套结构
-   物料名称唯一

------------------------------------------------------------------------

## 三、业务规则

1.  物料会新增
2.  物料会删除（逻辑删除）
3.  物料顺序会调整
4.  单位不统一（个 / 瓶 / 条 / 包 / 盒 等）
5.  第一个版本提交后不可修改
6.  分享页公开可查看
7.  一个门店一天可盘点多次

------------------------------------------------------------------------

## 四、数据库设计（最终版本）

### 1. categories（分类表）

``` js
{
  _id,
  name,
  sort_id,
  is_active,
  created_at
}
```

------------------------------------------------------------------------

### 2. materials（物料表）

``` js
{
  _id,
  name,
  category_id,
  unit,
  sort_id,
  is_active,
  created_at
}
```

排序规则：

-   按 category.sort_id 升序
-   按 material.sort_id 升序

------------------------------------------------------------------------

### 3. users（用户表）

``` js
{
  _id,
  openid,
  nickname,
  avatar,
  created_at
}
```

------------------------------------------------------------------------

### 4. stores（门店表）

``` js
{
  _id,
  user_id,
  store_code,
  store_name,
  created_at
}
```

一个用户可管理多个门店。

------------------------------------------------------------------------

### 5. inventories（盘点主表）

``` js
{
  _id,
  store_id,
  user_id,
  finished_at,
  created_at
}
```

------------------------------------------------------------------------

### 6. inventory_items（盘点明细表）

每个物料一条记录（快照存储）：

``` js
{
  _id,
  inventory_id,

  category_name_snapshot,
  category_sort_id,

  material_name_snapshot,
  material_sort_id,
  unit_snapshot,

  quantity
}
```

说明：

-   不保存 material_id
-   每次盘点为独立数据快照
-   历史数据不受结构变化影响

------------------------------------------------------------------------

## 五、Material Excel 模板

用于批量导入物料结构：

  ----------------------------------------------------------------------------------------
  category_name   category_sort_id   material_name   material_sort_id   unit   is_active
  --------------- ------------------ --------------- ------------------ ------ -----------
  分类A           1                  产品1           1                  个     1

  分类A           1                  产品2           2                  瓶     1

  分类B           2                  产品3           1                  包     1

  分类B           2                  产品4           2                  盒     1
  ----------------------------------------------------------------------------------------

规则说明：

-   material_name 不可重复
-   删除使用 is_active=0
-   排序通过 sort_id 控制

------------------------------------------------------------------------

## 六、填写页逻辑

展示顺序：

    分类A
      产品1
      产品2

    分类B
      产品3
      产品4

每个物料对应一个库存输入框。

------------------------------------------------------------------------

## 七、库存校验规则

允许：

-   数量为 0

禁止：

-   null
-   ""
-   未填写

正确校验逻辑：

``` js
if (value === "" || value === null) {
  return error
}
```

------------------------------------------------------------------------

## 八、提交流程

1.  遍历全部物料
2.  若存在未填写项 → 阻止提交
3.  弹出确认框：

```{=html}
<!-- -->
```
    库存提交后不可修改，是否确认提交？

4.  新建 inventory 记录
5.  批量写入 inventory_items

------------------------------------------------------------------------

## 九、分享页逻辑

分享参数：

    inventory_id

展示内容：

-   门店编号
-   门店名称
-   录入完成时间
-   所有物料库存数量

排序规则：

-   category_sort_id
-   material_sort_id

------------------------------------------------------------------------

## 十、未来升级方向

-   日消耗自动计算
-   异常库存预警
-   门店健康度评分
-   AI 进货建议
-   多门店对比分析

------------------------------------------------------------------------

## 十一、项目定位

这不是简单的盘点工具，而是：

> 幸运咖门店数据资产系统的底层基础设施。

当前版本为数据结构打基础，为后续数据智能化升级做准备。
